1) 제품 및 서비스의 목적·기능 및 특징

1-1. 제품 개발 배경

우리나라는 OECD 국가 중 산업재해 사망률이 여전히 상위권에 머무르고 있으며, 2022년 중대재해처벌법 시행 이후에도 중대산업사고가 지속 발생하고 있다. 그러나 구직 단계에서 기업의 안전성을 사전에 확인할 수 있는 수단은 사실상 전무하다. 구직자는 급여·복리후생 정보는 쉽게 비교하지만, 해당 기업의 산재 이력·중대재해 발생 여부·임금체불 명단 등재 사실 등은 알기 어렵다.

기업 안전 관련 정보는 이미 공공데이터로 존재한다. 근로복지공단의 산재처리현황, 고용노동부의 중대산업사고 발생현황과 임금체불 명단, 한국산업안전보건공단의 안전보건법령 등이 그것이다. 그러나 이들 데이터는 각기 다른 기관의 서로 다른 시스템에 분산되어 있어, 일반 구직자가 통합적으로 조회·비교하는 것은 현실적으로 불가능하다. 데이터는 있으나 활용되지 못하는 "정보 비대칭" 상태다.

1-2. 핵심 기능 — 자연어 한 문장으로 7개 도구 자율 호출

EconoBridge AI는 자연어 한 문장의 입력만으로 AI 에이전트가 7개 공공데이터 API를 자율적으로 호출하여 다음을 자동 수행한다.

  1. 채용정보 검색 — 워크넷 API로 조건에 맞는 채용공고 탐색
  2. 기업 산재이력 조회 — 근로복지공단·고용노동부 데이터로 산업재해 이력 확인
  3. 안전점수 산출 — 산재·중대재해·사망사고·임금체불 이력을 종합한 100점 만점 산출
  4. 위험기업 경고 — 안전점수가 낮은 기업은 명확히 경고 표시
  5. 안전한 일자리 추천 — 안전점수 기준 정렬된 맞춤 채용 결과 제공
  6. 훈련과정 연계 — HRD-Net 훈련과정 자동 추천
  7. 자기소개서 생성 — NCS 기반 맞춤 자기소개서 초안 자동 작성

1-3. 핵심 기술 — Claude Opus 4.7 ReAct 자율 에이전트

EconoBridge AI는 Anthropic Claude Opus 4.7(2026년 출시)을 메인 추론 엔진으로 채택하고, Vercel AI SDK 6의 ToolLoopAgent 위에서 ReAct(Reasoning + Acting) 패턴으로 동작한다. 사용자가 한 번 질문하면, 에이전트가 스스로 판단하여 최대 10단계까지 도구를 자율적으로 연쇄 호출한다. 단순 검색·요약을 넘어 "의사결정형 에이전트"로 작동하는 것이 본 서비스의 기술적 특징이다.

1-4. 안전점수 알고리즘 (핵심 차별 기능)

근로복지공단 산재처리현황과 고용노동부 중대산업사고 데이터를 교차 분석하여 100점 만점 안전점수를 산출한다. 이 알고리즘은 실제 코드(lib/tools/safety-check.ts)로 구현되어 작동한다.

2-2. 데이터 활용성 — "교차검증의 자동화"

기존 채용 플랫폼은 단일 데이터셋(채용공고)만 활용하지만, EconoBridge AI는 채용 + 산재 + 임금체불 + 훈련을 자율적으로 교차 호출한다. AI 에이전트가 채용공고에서 기업명을 추출하면, 동일 기업명을 키로 산재이력·임금체불 명단을 자동 조회하여 단일 응답에 종합한다.

핵심: 사람이 5~6개 사이트를 순회하며 일일이 비교하던 작업을, AI 에이전트가 자율적으로 단일 응답으로 대체한다. 이는 본 공모전에서 평가하는 "공공데이터 활용성" 의 본질에 부합한다.

2-3. 데이터 획득의 지속성 및 확장성

  • 지속성 — 「공공데이터의 제공 및 이용 활성화에 관한 법률」에 따른 정부 의무 공개 데이터로 중단 위험이 매우 낮다.
  • 활용범위 — 산재예방안전보건공단·국민연금공단·고용보험공단 데이터와 결합 시 취업–훈련–사회보장–안전관리의 통합 분석 으로 확장 가능하다.
  • 가공 가능성 — AI 에이전트의 도구 인터페이스는 Zod 스키마 기반의 JSON 표준화 를 채택하여, 신규 데이터 API 추가 시 도구 1개만 신설하면 에이전트가 자동으로 활용을 학습한다.

3-2. AI 자율성의 핵심 — ReAct 패턴 다단계 자율 워크플로

기존 챗봇이 "단일 쿼리 → 단일 응답"이라면, EconoBridge AI는 "단일 질의 → AI의 자율적 사고·도구 호출·결과 종합" 의 다단계 루프로 동작한다.

[사용자] "경기도에서 IT 웹개발자 채용 찾아줘" ↓ Step 1: 의도 분석 (Claude Opus 4.7) → 추출: 지역=경기, 직종=IT/웹개발, 장애여부=무 ↓ Step 2: search_worknet_jobs 호출 → 채용공고 5건 ↓ Step 3~7: check_company_safety × 5회 (각 기업 안전점수 자동 조회) ↓ Step 8: check_wage_violations (임금체불 교차 확인) ↓ Step 9: search_training_courses (역량갭 훈련 추천) ↓ Step 10: 결과 종합 (안전점수순 정렬 + 위험기업 경고)

이 모든 단계는 사용자의 추가 입력 없이 AI가 자율적으로 결정한다. Claude Opus 4.7의 향상된 도구 사용(Tool Use) 능력은 이러한 다단계 자율 워크플로의 안정성을 확보하는 핵심 기반이다.

3-3. AI 기술의 지속성 및 활용범위

  • 지속성 — Anthropic Claude API는 상업적 SLA를 제공하는 공식 API로 안정적 운영이 보장되며, Vercel AI SDK는 오픈소스로 벤더 락인 위험이 낮다.
  • 활용범위 — 본 ReAct 에이전트 아키텍처는 고용·복지·창업 등 정부 행정 영역의 다양한 자율 에이전트 서비스로 확장 가능하다. 예: 창업지원금 자동 매칭 에이전트, 복지급여 통합 신청 에이전트.
  • 모델 업그레이드 용이성 — 모델 ID 1줄 교체만으로 신규 모델 전환. 본 사업계획서 작성 시점에 최신 출시된 Claude Opus 4.7 로 업그레이드 완료 상태다.

3-4. AI 윤리 및 안전성

  • 데이터 출처 명시 — 모든 응답에 데이터 출처(기관명)를 명시하여 정보 책임성 확보
  • 자동 fallback — API 장애 시 캐싱·템플릿으로 fallback하여 가용성 보장
  • 검증 가능성 (Agent Trace) — AI의 모든 도구 호출 단계가 UI에서 시각화되어 추론 과정을 검증 가능
